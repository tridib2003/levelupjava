Constructors

  • A constructor initializes an object immediately upon creation, which make an object immediately usable upon its creation.

  • It has the same name as the class in which it is defined and is automatically called when an object of the class is created, before the new operator completes.

  • Constructors do not have any return type, not even void.

  • While allocating space in memory for an object, we use parentheses after the class name immediately after the new keyword, which essentially calls the constructor for the class.

  • When a constructor is not explicitly defined for a class, a default constructor is created. This default constructor sets all non-initialized variables to their default values. 0 for numeric, false for boolean, and null for reference types.

  • Once the user defines her own version of the constructor the default constructor is no longer used.

  • Parameterized constructors helps to initialize each object with the specified parameters at the time of object creation.
